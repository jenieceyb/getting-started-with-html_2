Basics of the Web & HTML

• The Web is:

- HTML is the glue of the web
○ Hyper Text Markup Language
○ Hyper Links
○ Main type of document on the web is HTML

- Major Pieces of the Web:

○ You
○ Computer & Browser: makes requests via the (internet). A program that runs on your computer to display files found on the web.
○ The Internet (the world's largest computer network) via (servers)
○ HTTP: the main protocol of the web
○ The Servers: then servers respond with files that the browser displays. The servers hosts files the make up the web.

• HTML: Hyper Text Markup Language

- HTML is made up of :
○ Text content (what you see)
○ Markup (what it looks like)
○ References to (e.g. images, videos etc.)
○ Other documents
○ Links to other pages

• Markup is made of things called TAGS (e.g. < name > contents < / name >)

○ Opening tag < name >
○ Contents
○ Closing tag < / name> (contains tag)

• B Tag = BOLD: < b >(no space)(content)(no space)< / b >
e.g. HTML is reasonably straightforward

• Em- Emphasis = Italics (e.g. content)
○ I am going to learn to love HTML.

•HTML Attributes:

○ < tag ATTR="value">content< /tag>
○ Anchor= < a> for Making Links
○ < a (opening anchor) (space) href (attribute name)="www.reddit.com" (value)>derp (contents)(closing anchor)
○ I am going to learn to love < a href="www.w3c.org/html">HTML. = I am going to learn to love HTML.
○ This website is < a href="www.udacity.com">my favorite = This website is my favorite

• Images = < img>
○ < img src="url" alt="text(required)>
○ Images don't need a closing tag because they have no content ○ e.g. < img src="http://www.udacity.com/cs253x/hipmunk.png" alt="hipmunk"> =

• Whitespace
○ Text on multiple lines = < br> break < void>
○ P = < p>content< /p>

• Inline vs. Block

- < br> = inline: ending a line, then wraps to the next line
- < p> = block: creates an invisible box around the text. It can have height and width.

• Span/Div: both are normal elements, both can have content. They don't do anything other than contain their content. There is a way to attach styles to them/adjust different behaviors to
how they display.
- Span= inline < span>content
- Div= block < div>content
- Add attributes:
○ < span class="foo">content CSS class (styles/different looks)
- a, span, br, img, strong = Inline
- Div, p, form = Block
